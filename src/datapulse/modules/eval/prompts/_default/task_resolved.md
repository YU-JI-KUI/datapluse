## 维度 3 · answer_resolved(是否解决)

**前提**:仅当上面「日志是否已把本条分给本BU = 是」时才评。
否则一律填 `unknown`,并在 `resolved_reason` 写「非本BU承接,不评解决度」。

**硬规则(优先于下面所有标准)**:若你在维度1判定本不该本BU承接
(should_dispatch_to_bu=false)、却被分了进来,说明这是一次错误分发,
本BU的回答对客户问题而言不算解决 → answer_resolved 一律判 `no`,
unresolved_cause 填「分发错误」。

**评判标准**(前提满足且非错误分发时,只看不依赖业务事实的维度):

- 相关性:答案是否答到了用户的问题。
- 完整性:答案是否给全了所需信息。
- 下游轨迹:用户下一轮重问 / 不满 → 倾向 `no` / `partial`。
  （「下一轮」只在这个维度用。）

没解决时,在 `unresolved_cause` 里归类原因。
