# 答案样例库

每个解析器对应一个样例文件，存**真实原始答案** + 结构判据 + 期望解析结果。
看代码时对照，一眼知道某个解析器处理的是什么形态的答案。

## 命名约定

文件名 = 解析器的 `name`：`<bu>.<结构>.md`，如 `securities.robot_menu.md`。

## 现有样例

| 文件 | 解析器 | 说明 |
|---|---|---|
| `generic.msgcontext_card.md` | MsgContextCardParser | 通用·渲染卡 msgContext.msgInfo（寿险/证券日志主体，data.content） |
| `generic.content_data.md` | ContentDataParser | 通用·文本回复 content_data |
| `generic.llm_api.md` | LlmApiRespParser | 通用·LLM API 响应 msg |
| `generic.jump_platform.md` | JumpPlatformParser | 通用·跳端卡 crossCardType=JUMPPLATFORM（拒识后跨 App 跳转） |
| `generic.benefit_card.md` | BenefitCardParser | 通用·权益领取结果卡 catalogId+benefits（含寿险活动权益卡 activity-mx-benefit-card） |
| `generic.agreement_card.md` | AgreementCardParser | 通用·协议同意卡 agreements（协议弹窗） |
| `generic.service_nav.md` | ServiceNavCardParser | 通用·服务导航卡 tabs+funcList（如「平安救急服务／7X24小时全天候紧急救援服务」，只取标题+副标题） |
| `life.faq_card.md` | LifeFaqCardParser | 寿险·金管家 FAQ 知识库卡 card_content.data（正文 + 关联问题 options） |
| `life.policy_list_card.md` | LifePolicyListCardParser | 寿险·金管家保单列表卡 policyInfos（抬头 + 各保单险种名+投保日） |
| `life.service_flow_card.md` | LifeServiceFlowCardParser | 寿险·金管家服务流程卡 gbdData.oneKeyServiceName（为您转接【服务名】服务） |
| `life.clarify_card.md` | LifeClarifyCardParser | 寿险·金管家意图不明澄清卡 data.msg+options（澄清话术 + 推荐问题） |
| `life.multi_round_card.md` | LifeMultiRoundCardParser | 寿险·金管家多轮文本卡 data.answer+capsule（intentCode=COMMON_MULTIPLE，正文 + 胶囊追问，如「退保」） |
| `life.faq_answer_card.md` | LifeFaqAnswerCardParser | 寿险·金管家 FAQ 答案卡 data.answerList+rankType=FAQ（H5 访客，多段 HTML 正文按段落拆行，如「门店信息查询」） |
| `securities.xiaoan_card.md` | XiaoAnCardParser | 证券·小安特有卡（同花顺 thsData / 列表卡片 list） |
| `securities.robot_menu.md` | RobotMenuItemsParser | 证券·菜单卡 robotMenuItems（问题在 msgContent.menuItems） |
| `securities.robot_text_answer.md` | RobotTextAnswerParser | 证券·关联问卡 robotTextAnswer（问题在 msgInfo.relatedQuestions） |
| `securities.search_card.md` | SearchCardParser | 证券·搜索返回卡 searchData.searchType（导向搜索/快捷服务入口，query 各异，统一净化成「搜索返回」，配 `*` 短路规则判已解决） |

## 新增解析器时

内网新写一个 `AnswerParser` 子类后，把那条真实答案样例照上面格式存一个
`<解析器name>.md` 到本目录，保留原文与期望输出，方便日后排查与回归。
