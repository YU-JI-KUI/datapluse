# 调优与重跑

评测出报告后，通常需要根据结果纠偏或调优。不同操作对指标的影响方式不同，本页说明其规律与用法。

## 核心规律

::: tip 一条规律
改「判定规则」类的内容（提示词、业务知识、业务分类、规则短路、活动标问）只影响下次评测，已有报告不会自动变化，需重跑才反映。

唯一例外是人工复核——它是叠加层，实时改指标，无需重跑。
:::

## 四种操作对比

| 操作 | 位置 | 调模型 | 落库 | 指标变化 | 生效时机 |
|------|------|--------|------|---------|---------|
| 单条试跑 | 详情抽屉 | 是（1 条）| 否 | 不变 | 即时预览，仅验证提示词改得对不对 |
| 人工复核 | 详情抽屉 | 否 | 是 | 增量重算 | 立即，重进结果页即新指标 |
| 勾选重跑 | 明细表 | 是（选中）| 是（覆盖）| 全量重算 | 后台异步，完成后重进结果页，跳过已复核，上限 2000 条 |
| 全量重测 | 历史 / 结果页 | 是（全部）| 是（清空重跑）| 全部重评 | 耗时较长 |

## 人工复核

改一条的最终结论，实时改指标。

在详情抽屉的「人工复核」区，修改分发是否正确、是否解决、业务分类，并写评论后提交。指标按最终值（人工优先、大模型兜底）增量重算：

- 分发：复核值与大模型不同时，判对数相应加减一。
- 解决：仅对分到本 BU 的样本生效，复核值与大模型不同时，已解决数相应加减一。
- 需复核：被复核的行从需复核数减一。

无需重跑，重进结果页指标即更新。撤销复核可还原。

## 调优业务知识

改判定规则，让大模型判得更准。业务人员在详情抽屉里完成闭环：

1. 发现大模型判错，在「调优业务知识」区当场编辑本 BU 的业务知识（补充规则，如「含'已为您转接人工'的答案视为已解决」）。
2. 试跑：用编辑框里尚未保存的临时内容重评这一条，对比新旧结果。不落库、不影响指标。
3. 不满意继续改，满意后保存业务知识（写入库，整个 BU 共享）。
4. 保存不等于指标变化。要让业务知识作用到已有报告，需勾选重跑相关行，或全量重测。

## 提示词管理

`/eval/prompts` 各槽位影响不同维度，改后下次评测生效，已有报告需重跑。

| 槽位 | 影响 |
|------|------|
| judge_system | 大模型的判定立场与总体设定 |
| task_dispatch | 分发判定，影响分发准确率 |
| task_business_type | 业务分类打标，影响分类分布 |
| task_resolved | 是否解决，影响解决率 |
| task_review | 需复核判定，影响需复核数 |
| business_knowledge | 业务方补充规则，业务人员改这里即可 |

::: tip 快照隔离
评测发起时会快照当前提示词，整个任务用同一份。跑到一半有人改提示词不会污染进行中的任务，只影响下次评测。
:::

## 典型工作流

**首次评测**

1. 在「评测」页选 BU、上传日志 Excel（或用样例），发起评测。
2. 在「历史评测」看进度，完成后进入结果页看指标。

**发现指标异常 → 定位 → 调优**

1. 结果页看来源分布、分类分布、轮次分布，找出异常。
2. 明细表筛选「需复核」或某分类，点开详情抽屉看判错在哪。
3. 判错分两种：
   - 个别条判错：直接人工复核纠正，实时改指标。
   - 一类系统性判错：详情抽屉编辑业务知识 → 试跑验证 → 保存 → 明细表勾选相关行重跑（或全量重测）。

**节省模型调用**

- 结果确定的高频问答（如「转人工」）在「规则短路」配规则，问题与答案匹配即免调模型，仍计指标。
- 前端固定按钮的固定回复（如「帮我解锁消费权益」）在「活动标问」配置，整条跳过、不计指标。

**导出**

结果页可导出评估报告（多 sheet 概览）、逐条明细、不一致 case 三种 Excel，文件名带上传文件名前缀。大数据量下有加载态，防止重复点击。

## 稳定性保障

- **多 POD 调度**：任务状态存于数据库，多副本部署下抢占式领取任务、全局串行执行、心跳续约、崩溃自动接管续跑。
- **断点续跑**：任务中断后只补未完成的行，已落盘结果不重做。
- **限流退避**：触发模型限流时自动暂停、退避后续跑，一批中已成功的先落盘。
- **容错解析**：模型偶发返回非法 JSON 时升温重试并容错解析。
