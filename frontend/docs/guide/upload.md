# 数据上传

进入左侧导航「数据上传」页面，支持拖拽或点击上传文件。

## 支持的文件格式

| 格式 | 扩展名 | 文本列识别规则 |
|------|--------|----------------|
| Excel | `.xlsx` | 优先匹配列名：`text` / `content` / `文本` / `内容` / `query` / `问题` / `input`，未匹配时取第一列 |
| CSV | `.csv` | 识别规则同 Excel |
| JSON | `.json` | 支持字符串数组 `["文本1", "文本2"]`，或对象数组 `[{"content": "..."}]` |

::: tip 带标注的历史数据上传
如果文件中同时包含文本和标签列（如 `label` / `标签` / `意图`），系统会识别标签列并自动创建预标注记录，跳过 LLM 预标注步骤，直接进入人工审核。适用于导入已有标注的历史数据集。
:::

## 自动去重

上传时系统会对文本内容做 SHA-256 哈希，自动跳过以下两类重复：

- **跨文件重复**：与数据库中已有数据完全相同的文本
- **文件内重复**：同一文件内多次出现的相同文本

上传完成后的提示弹窗会显示：
- **新增 N 条**：成功入库的新数据
- **跳过 M 条**：无效文本，以下情况会被跳过：空行或纯空白、长度 < 2 个字符（过短无意义）、长度 > 512 个字符（过长）
- **去重 K 条**：内容重复被跳过的数量（跨文件重复 + 文件内重复都计入）

::: info
去重基于文本内容本身（规范化后），与文件名无关。同一批文本分多次上传，只有第一次会入库，后续上传全部被去重跳过。
:::

## 数据管理

上传完成后，切换到 **数据管理** 标签页可以查看已入库的数据列表，支持按状态筛选、搜索，以及逐条删除。
