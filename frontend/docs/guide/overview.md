# 平台简介

Datapulse 是一个面向 NLP 任务的 AI 数据生产平台，覆盖从原始数据上传到高质量标注集导出的完整流程，主要应用于意图识别、文本分类等场景。

## 整体架构

```
原始数据上传
    ↓
Pipeline 自动处理
  ├── 数据清洗（规范化文本）
  ├── LLM 预标注（自动预测标签）
  └── 冲突检测（质量校验）
    ↓
人工标注工作台
  ├── 标注员逐条确认 / 修正
  └── 管理员裁决冲突
    ↓
高质量数据导出（用于模型训练）
```

## 角色说明

平台设计了两类角色，权限各有不同。

**管理员（admin）**：可以访问所有功能，包括数据集管理、用户管理、配置中心、冲突裁决等。

**标注员（annotator）**：只能访问被分配的数据集，主要使用标注工作台完成标注任务，无法修改系统配置。

## 数据状态流转

每条数据在系统中有明确的状态，随处理进度自动推进：

| 状态 | 含义 |
|------|------|
| `raw`（原始）| 刚上传，尚未清洗 |
| `cleaned`（已清洗）| 经过文本规范化，等待预标注 |
| `pre_annotated`（预标注）| 模型已预测标签，等待人工确认 |
| `annotated`（已标注）| 至少一名标注员完成标注 |
| `checked`（已检测）| 通过冲突检测，数据质量最高 |
