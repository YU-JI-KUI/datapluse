# 冲突检测与裁决

冲突检测是 Pipeline 的第三个步骤，对已完成人工标注的数据进行质量校验，找出存在分歧的条目并交由标注员裁决。

## 冲突类型

### 标签冲突

多名标注员对同一条文本给出了不同的意图标签。

例如，标注员 A 将某句话标为 `query_balance`，标注员 B 标为 `query_transaction`，两者不一致，系统将此条目标记为标签冲突。

### 语义冲突

两条文本的语义高度相似（向量余弦相似度超过阈值），但被标注为不同的意图标签。

例如「我要转账」和「帮我转一笔钱」语义接近，若一条标为 `transfer_money` 另一条标为 `other`，可能存在标注不一致的问题。

::: info 语义冲突依赖向量化
语义冲突检测需要先完成向量化（Embedding）。若尚未向量化，Pipeline 只检测标签冲突，跳过语义冲突部分。向量化完成后可单独重跑冲突检测。
:::

## 查看冲突列表

进入左侧导航「冲突检测」页面，可以看到：

- 所有待裁决的冲突条目
- 每条的冲突类型（标签冲突 / 语义冲突）
- 参与标注的人员及各自给出的标签
- 文本内容

可以按冲突类型筛选，分批处理。

## 裁决冲突

标注员点击冲突条目，在右侧面板中：

1. 查看各标注员的标签和评论
2. 选择正确的最终标签（或输入新标签）
3. 可以添加裁决备注
4. 点击「确认裁决」提交

裁决后，该条目的最终标签固定为裁决者选择的结果（`label_source = manual`），状态更新为 `checked`，不再触发冲突流程。

::: warning 裁决不可撤销
冲突裁决一旦提交，视为最终结论。如需修改，请在数据探查页面手动调整。
:::

## 批量裁决

若有大量相同类型的冲突（如同一意图的标注员分歧），可以勾选多条冲突条目，选择统一的标签，点击「批量裁决」一次性处理。

## 单独触发冲突检测

如果只需要重新运行冲突检测（而不重跑整个 Pipeline），可以点击页面右上角的「检测冲突」按钮。这在以下场景中很有用：

- 补充了新的标注，想立即检查冲突
- 向量化刚完成，需要补充语义冲突检测

## 数据状态说明

| 状态 | 含义 |
|------|------|
| `annotated` 且有冲突标记 | 等待标注员裁决 |
| `checked` | 已通过冲突检测（无冲突或已裁决），可放心导出 |
