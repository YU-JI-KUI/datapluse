# LLM 预标注

预标注是 Pipeline 的第二个步骤，由配置的大语言模型自动为每条文本预测一个意图标签，并给出置信度分数，供标注员参考。

## 预标注的作用

人工标注每条数据都需要判断意图类别，成本较高。预标注让模型先做一遍，标注员只需核对模型的结论是否正确：

- 置信度高时直接点击确认，几秒内完成一条
- 置信度低或预测明显错误时手动选择正确标签

实践中，高质量的预标注可以将标注速度提升 3～5 倍。

## 查看预标注结果

进入左侧导航「预标注」页面，可以看到：

- 所有已完成预标注的数据列表
- 每条数据的预测标签和置信度分数（0～1）
- 当前使用的模型名称

可以按置信度范围筛选，快速找到低置信度或高置信度的数据。

## 触发预标注

通常预标注由 Pipeline 自动触发。如果需要单独重跑（例如更换了模型配置），点击页面右上角的「运行预标注」按钮，只对 `cleaned` 状态的数据重新预测。

::: info 不会覆盖已标注数据
预标注只处理尚未进入人工标注环节的数据，已被标注员提交过标注的数据不会被重新预标注。
:::

## 触发向量化

点击「触发向量化」按钮，可以手动启动向量化（Embedding）和 FAISS 索引构建任务。向量化完成后，冲突检测步骤才能进行语义相似度分析。

向量化是计算密集型操作，在后台运行，不影响其他功能的使用。进度显示在 Dashboard 右侧状态面板的「离线任务」区域。

## 置信度参考标准

| 置信度范围 | 建议操作 |
|-----------|---------|
| ≥ 0.90 | 预测通常可靠，可直接确认 |
| 0.70 ～ 0.89 | 仔细核对文本含义后再确认 |
| < 0.70 | 建议手动判断，不依赖模型结论 |

::: tip
置信度只是参考，最终标注由标注员决定。遇到模型明显错误的情况，直接选择正确标签即可。
:::

## 模型配置

使用的模型在「配置中心」中设置，仅管理员可操作。更换模型后，对已有数据的预标注记录不会自动清除，需要重新运行预标注才会用新模型覆盖旧结果。
