# 数据集管理

数据集是系统的核心工作单元，所有数据、标注记录、Pipeline 状态都按数据集严格隔离。此页面仅管理员可访问。

## 基本操作

### 新建数据集

点击右上角「新建数据集」，填写名称（必填）和描述（可选），点击「创建」。

创建完成后数据集状态默认为**活跃**，标注员需要被分配后才能访问。

### 分配用户

点击数据集行的 👥 图标，在弹窗中勾选需要访问此数据集的标注员账号，点击「保存分配」。

::: info
管理员无需分配，默认可访问所有数据集。分配仅针对普通标注员账号。
:::

### 编辑数据集

点击 ✏️ 图标，可修改名称、描述，也可以将数据集切换为**停用**状态（停用后标注员无法选择该数据集）。

### 重置 Pipeline 状态

点击 🔄 图标（橙色），可将该数据集的 Pipeline 状态重置为**空闲**。

这个操作只清除状态记录，**不会修改任何数据**。适用于以下场景：

- Pipeline 因服务重启等原因卡在「运行中」状态
- 需要重新触发 Pipeline，但旧状态阻止了启动
- 向量化任务（embed job）失败后需要清除错误重试

弹窗中有一个「同时重置向量化状态」的选项，默认不勾选（只重置主流程状态，向量化进度保留）。

### 删除数据集

点击 🗑️ 图标，确认后将**永久删除**该数据集及其全部关联数据（数据条目、标注记录、冲突等），不可恢复。

::: warning
删除操作不可撤销，执行前请确认数据已导出备份。
:::

## 数据集切换

左侧侧边栏顶部有数据集选择器，点击可以切换当前工作的数据集。切换后所有页面会自动刷新，展示新数据集的数据。
