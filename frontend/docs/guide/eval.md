# 评测概述

AI 评测模块用于评估对话系统的回答质量。上传一份对话日志 Excel，系统对每条客户问题调用大模型做一次结构化判定，汇总成一份包含分发准确率、问题解决率、业务分类分布等指标的评测报告。

评测与标注平台相互独立，自成体系：上传日志 → 逐条评测 → 出报告 → 调优重跑，不依赖数据集与标注流程。

## 整体流程

```
上传对话日志 Excel
    ↓
逐条处理每个样本
  ├── 活动标问：整条跳过，不评测、不计入指标
  ├── 规则短路：命中写死结果，免调大模型，但计入指标
  └── 其余样本：调用大模型逐条判定
    ↓
汇总评测报告（分发准确率 / 问题解决率 / 分类分布 …）
    ↓
调优（人工复核 / 业务知识 / 提示词）→ 重跑验证
```

## 核心概念

| 概念 | 说明 |
|------|------|
| 业务单元（BU）| 评测的业务线，如证券、寿险。业务分类、规则、提示词都按 BU 各存一套 |
| Judge | 大模型对单条问答的结构化判定，输出是否本 BU 承接、业务分类、是否解决、是否需复核等字段 |
| 样本 | 日志中的一条「客户问题 + 系统答案」，评测的最小单位。一通多轮对话含多条样本 |
| 分发 | 判断某条问题是否应由本 BU 承接，与日志中「实际分给了谁」比对 |
| 金标 | 人工事先在 Excel 中标注的「是 / 否」列。有金标进入校准模式，无金标进入生产模式 |
| 活动标问 | 前端固定按钮触发的固定回复（如「帮我解锁消费权益」），不经大模型，评测时整条跳过 |
| 规则短路 | 结果确定的问答配好写死判定（如「转人工」），命中即免调大模型，但仍计入指标 |
| 业务知识 | 业务方补充的判定规则，一段自然语言，拼入 Judge 提示词让判定更贴合业务 |
| 人工复核 | 人工覆盖大模型的某条判定，指标按「人工优先」实时重算，无需重跑 |

## 评测模式

系统根据上传的 Excel 是否含金标列自动判断模式：

- **校准模式**：Excel 含「分发是否正确」或「答案是否解决客户问题」列（填「是 / 否」）。除常规指标外，额外计算 κ、F1 等一致性指标，用于验证大模型判定是否可信。
- **生产模式**：无金标列。分发准确率、问题解决率等指标仍可计算（以日志客观事实为准），但没有一致性校验。

## 管理入口

侧边栏「AI 评测」下有六个入口：

| 菜单 | 用途 | 改后生效时机 |
|------|------|-------------|
| 评测 | 上传 Excel 或选样例，发起评测 | — |
| 历史评测 | 任务列表、进度、进入结果页 | — |
| 业务分类 | 维护本 BU 的业务分类清单 | 下次评测；旧报告需重跑 |
| 活动标问 | 维护要整条跳过的问题 | 下次评测；旧报告需重跑 |
| 规则短路 | 维护「问题 + 答案 → 写死判定」规则 | 下次评测；旧报告需重跑 |
| 提示词管理 | 编辑 Judge 各段提示词与业务知识 | 下次评测；旧报告需重跑 |

## 上传要求

Excel 列名按「包含匹配」识别，缺必需列会直接报错。

**必需列**：应用会话 ID、客户咨询轮次、客户问题、答案、分发 BU。

**金标列（可选）**：分发是否正确、答案是否解决客户问题。填「是 / 否」，决定是否进入校准模式。

::: warning 分发 BU 是必需列
分发准确率、问题解决率都以「日志实际分给谁」为事实依据。缺少该列会导致核心指标全部失真，因此缺列时直接报错、不放行。
:::

## 处理三阶段

**数据加载**：读取 Excel，过滤活动标问（客户问题与清单精确相等即整条跳过，仅作后续轮上下文保留）。评测样本数 = 日志总条数 − 活动标问数。

**逐条评测**：先过规则短路（客户问题与答案都精确匹配规则时，用写死结果、不调大模型，仍计入指标）；未命中则调用大模型判定。业务分类采用闭集约束——清单外的自创分类统一归入「非 XX 业务」兜底桶，报告中的业务分类恒为「配置的 N 个 + 非 XX 业务」。

**汇总指标**：逐条结果落盘后汇总成报告。各指标口径见[指标口径](/guide/eval-metrics)。
