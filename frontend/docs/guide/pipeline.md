# Pipeline 流程

Pipeline 是系统自动处理数据的核心流程，三个步骤依次执行，将原始数据转化为可标注的高质量样本。

## 流程概览

```
raw（原始）
  ↓ 数据清洗
cleaned（已清洗）
  ↓ LLM 预标注
pre_annotated（预标注完成）
  ↓ 冲突检测
annotated / checked
```

## 步骤说明

### 1. 数据清洗

处理所有状态为 `raw` 的数据，对文本内容做基础规范化：

- 去除首尾多余空白
- 将连续空格、换行、制表符压缩为单个空格
- 删除不可见的 ASCII 控制字符

处理完成后状态更新为 `cleaned`，等待预标注。

### 2. LLM 预标注

调用配置的大语言模型，对所有 `cleaned` 状态的数据自动预测意图标签，并给出置信度分数。

- 预标注结果保存在系统中，供标注员在标注工作台参考
- 置信度分数范围 0～1，越高表示模型越确信
- 预标注不是最终结论，标注员可以修改

处理完成后状态更新为 `pre_annotated`。

### 3. 冲突检测

对所有已完成人工标注的数据（`annotated` 状态）进行质量校验，检测两类冲突：

- **标签冲突**：多名标注员对同一文本给出了不同的标签
- **语义冲突**：两条文本语义高度相似，但被标注为不同标签（需要向量化完成后才能检测）

无冲突的数据状态更新为 `checked`；有冲突的保持 `annotated` 并标记冲突类型，等待管理员裁决。

::: info 关于语义冲突检测
语义冲突检测依赖向量化数据。如果尚未完成向量化，Pipeline 仍会正常执行，只是跳过语义冲突部分，只检测标签冲突。向量化完成后可以单独重跑冲突检测步骤。
:::

## 如何触发

进入 **Dashboard**，点击右上角「运行 Pipeline」。Pipeline 在后台运行，不会阻塞界面操作。

**单步触发**：如果只需要重新执行某一个步骤，可以在对应功能页面单独触发：
- 预标注页面 → 「运行预标注」按钮
- 冲突检测页面 → 「检测冲突」按钮

## Pipeline 状态说明

| 状态 | 含义 |
|------|------|
| 空闲 | 未运行或上次已完成正常 |
| 运行中 | 正在执行，Dashboard 显示实时进度和速度 |
| 已完成 | 本次运行成功，可查看各步骤结果摘要 |
| 出错 | 某步骤执行失败，错误信息显示在 Dashboard 状态面板 |

::: warning 状态卡住时
若 Pipeline 长时间显示「运行中」但实际已停止（如服务重启后），可前往「数据集管理」→ 对应数据集 → 点击橙色重置按钮，清除卡住状态后重新触发。
:::

## 向量化（离线任务）

向量化（Embedding）和 FAISS 索引重建是计算密集型操作，已从主流程中解耦，作为独立的离线任务运行。

- 在 **预标注** 页面点击「触发向量化」按钮手动启动
- 向量化在后台运行，不影响 Pipeline 其他步骤
- Dashboard 右侧状态面板的「离线任务」区域显示向量化进度

向量化完成后，冲突检测才能进行语义冲突分析。
